# Components

Charlie UI ships 30 components organized into four categories. Each component is fully typed, supports `className` overrides, and follows consistent API patterns.

## Atoms

Small, foundational building blocks used across the library.

| Component | Description |
|---|---|
| [Badge](badge.md) | Inline status labels with color variants |
| [Button](button.md) | Primary interactive element with 5 variants and 3 sizes |
| [Divider](divider.md) | Horizontal rule with solid or dotted styles and optional label |
| [Input](input.md) | Text input with label, helper text, error state, and icon slots |
| [Kbd](kbd.md) | Keyboard shortcut display element |
| [Skeleton](skeleton.md) | Loading placeholder in text, circle, or rectangle shapes |
| [Toggle](toggle.md) | Animated on/off switch |

## Cards

Content containers with structured layouts.

| Component | Description |
|---|---|
| [Card](card.md) | General-purpose card with Header, Body, and Footer sub-components |
| [BlogCard](blog-card.md) | Article preview card with image, tag, title, excerpt, and date |
| [FeatureCard](feature-card.md) | Feature showcase card with icon, title, description, and optional image |
| [PricingCard](pricing-card.md) | Pricing tier card with features list and CTA |
| [SocialCard](social-card.md) | Gradient-backed card for social links |
| [Testimonial](testimonial.md) | Customer quote card with author info and avatar |

| [Hero](hero.md) | Page hero section with centered or split layout |
